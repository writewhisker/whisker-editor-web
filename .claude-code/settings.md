# Claude Code Settings for whisker-editor-web

## Planning and Phase Documents

**IMPORTANT:** Planning documents, phase documents, and completion reports should NOT be created in this repository (whisker-editor-web).

All planning, phase plans, completion reports, and implementation tracking documents should be stored in the private repository:
- **Repository:** `whisker-implementation`
- **Location:** `/planning/` directory

This keeps the public repository clean and focused on code, while maintaining detailed implementation tracking privately.

### Document Types to Store in whisker-implementation:
- Phase plans (`PHASE_*_PLAN.md`)
- Completion reports (`PHASE_*_COMPLETION.md`)
- Implementation decisions (`*_DECISION.md`)
- Migration plans (`*_MIGRATION_PLAN.md`)
- Progress reports (`*_REPORT.md`)
- Any other planning or tracking documents

### Documents that SHOULD remain in whisker-editor-web:
- README.md
- CONTRIBUTING.md
- LICENSE
- Technical documentation (API docs, architecture docs)
- User-facing documentation
